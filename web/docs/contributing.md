# Contributing

Velarix accepts contributions across the API, reasoning engine, SDKs, integrations, benchmark harness, and documentation.

## Contribution Flow

1. fork the repository
2. clone locally
3. create a branch for the change
4. implement the change with tests where appropriate
5. open a pull request against `main`

## Coding Standards

- Follow the existing code style.
- Write tests for any new features or bug fixes.
- Ensure the relevant build and test paths pass before submission.

## Reporting Issues

Use GitHub issues for bugs, regressions, and feature requests.
