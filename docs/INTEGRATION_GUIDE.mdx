# Integration Guide

Velarix integrates at the execution boundary.

The primary pattern is consistent across products and frameworks:

1. collect the facts that justify the action
2. derive the recommendation
3. create the decision
4. call `execute-check` immediately before the side effect
5. execute only if the decision remains valid

## Primary Integration Surfaces

- Python SDK for direct session, fact, slice, and decision workflows
- OpenAI adapter for model-facing observation and slice injection
- LangGraph checkpointer backed by Velarix session history
- CrewAI memory helper for query-aware belief injection
- LlamaIndex retrieval helper for current valid facts
- `vlx` CLI for health, slice, review, invalidation, retraction, export, and benchmark flows

## Python SDK

```python
from velarix import VelarixClient

client = VelarixClient(base_url="http://localhost:8080", api_key="dev-admin-key")
session = client.session("payment_approval_001")

session.observe("vendor_verified", {"vendor_id": "vendor-17"})
session.observe("invoice_approved", {"invoice_id": "inv-1042"})
session.observe("budget_available", {"cost_center": "ENG-01"})

decision = session.create_decision(
    "payment_release",
    fact_id="decision.release_payment",
    subject_ref="inv-1042",
    target_ref="vendor-17",
    dependency_fact_ids=["vendor_verified", "invoice_approved", "budget_available"],
)

check = session.execute_check(decision["decision_id"])
```

## Query-Aware Slice Pattern

Use slices when an agent needs current belief state, not a full session dump:

```python
slice_markdown = session.get_slice(
    format="markdown",
    query="payment approval blockers vendor verification",
    strategy="hybrid",
    include_dependencies=True,
    max_facts=40,
)
```

This is the preferred pattern for both large-context models and smaller context windows.

## LangGraph

`VelarixLangGraphMemory` checkpoints LangGraph state into Velarix session history.

That keeps graph state, auditability, and belief retrieval inside the same session boundary.

## CrewAI

`VelarixCrewAIMemory` injects the current ranked belief slice into a CrewAI task for a specific query.

That keeps prompt context narrow while preserving the facts most relevant to the task.

## Stateful And Stateless Use

Velarix sessions are stateful by design.

Stateless use is supported by creating a fresh session per request or mission.

## Product Guidance

Velarix is strongest when it remains close to the final execution step.

If an integration is far from the side effect, it is secondary to the main product flow.
